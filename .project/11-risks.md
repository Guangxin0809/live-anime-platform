# 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 支付集成复杂，回调调试困难 | 支付流程卡住 | 预留支付模拟模式（sandbox），先跑通流程再接入正式环境 |
| 动漫内容版权合规 | 法律风险 | 设计时预留版权信息字段，内容审核流程 |
| 国内网络访问 npm 包慢 | 开发效率 | 配置 .npmrc 使用淘宝镜像源 |
| 微信/支付宝审核周期长 | 发布延迟 | 提前注册开发者账号，沙箱环境先行联调 |
| Docker 镜像构建慢 | CI/CD 效率 | 合理分层利用缓存，dev 依赖和生产依赖分离 |
| Alpine 下 Prisma 依赖二进制兼容 | 容器启动失败 | 明确 `prisma generate` 在镜像内执行，使用 `node:20-alpine` 官方镜像保证 glibc 兼容 |
